# gin-auth

A JWT authentication REST API built with Go, Gin, sqlc, and PostgreSQL.

---

## Project structure

```
gin-auth/
├── cmd/
│   └── server/
│       └── main.go                        # Entry point — wires all dependencies
├── config/
│   └── config.go                          # Loads config from environment variables
├── api/
│   └── routes.go                          # Route registration
├── internal/
│   ├── database/
│   │   └── postgres.go                    # pgx connection pool
│   ├── model/
│   │   └── user.go                        # Domain structs + request/response types
│   ├── repository/
│   │   └── user_repository.go             # Data access layer (uses sqlc)
│   ├── service/
│   │   └── auth_service.go                # Business logic (register, login, JWT)
│   ├── handler/
│   │   └── auth_handler.go                # HTTP handlers
│   └── middleware/
│       └── jwt.go                         # JWT auth middleware
├── pkg/
│   └── response/
│       └── response.go                    # Shared JSON response envelope
├── db/
│   ├── migrations/
│   │   ├── 001_create_users.up.sql        # Create users table
│   │   └── 001_create_users.down.sql      # Drop users table
│   ├── queries/
│   │   └── users.sql                      # SQL queries (input for sqlc)
│   └── sqlc/                              # Auto-generated by sqlc — do not edit
│       ├── db.go
│       ├── models.go
│       └── users.sql.go
├── go.mod
├── sqlc.yaml                              # sqlc configuration
└── .env.example
```

## Quick Start

Choose your preferred way to run the project:

### Option 1: Run with Docker (Recommended)

```bash
# Start all services (PostgreSQL, migrations, and app)
make docker-up

# View logs
make docker-logs

# Stop everything
make docker-down

# Reset database and restart
make docker-clean
make docker-up
```

The app will be available at `http://localhost:8080`

### Option 2: Run Locally

```bash
# Install dependencies
go mod tidy

# Configure environment
cp .env.example .env
# Edit .env with your values

# Start PostgreSQL and run migrations
brew services start postgresql  # or: sudo systemctl start postgresql
make migrate-up

# Run the server
make run
```

The app will be available at `http://localhost:8080`

## Step-by-step setup

### Step 1 — Prerequisites

Make sure you have these installed:

```bash
# Check Go (need 1.21+)
go version

# Check PostgreSQL
psql --version

# Install sqlc
go install github.com/sqlc-dev/sqlc/cmd/sqlc@latest

# Verify sqlc is available
sqlc version
```

---

### Step 2 — Clone / create the project folder

```bash
mkdir gin-auth
cd gin-auth
```

Copy all the project files into this folder following the structure above.

---

### Step 3 — Install Go dependencies

```bash
go mod tidy
```

This downloads all packages listed in `go.mod`:
- `github.com/gin-gonic/gin` — HTTP framework
- `github.com/golang-jwt/jwt/v5` — JWT signing and validation
- `github.com/jackc/pgx/v5` — PostgreSQL driver
- `golang.org/x/crypto` — bcrypt for password hashing

---

### Step 4 — Set up PostgreSQL

```bash
# Start PostgreSQL if it isn't running (macOS with Homebrew)
brew services start postgresql

# Or on Linux
sudo systemctl start postgresql

# Create the database
psql -U postgres -c "CREATE DATABASE gin_auth;"
```

---

### Step 5 — Run the database migration

```bash
psql -U postgres -d gin_auth -f db/migrations/001_create_users.up.sql
```

Verify the table was created:

```bash
psql -U postgres -d gin_auth -c "\d users"
```

You should see:

```
                        Table "public.users"
   Column   |            Type             | Nullable |      Default
------------+-----------------------------+----------+-------------------
 id         | bigint                      | not null | nextval(...)
 username   | character varying(32)       | not null |
 email      | character varying(255)      | not null |
 password   | text                        | not null |
 created_at | timestamp with time zone    | not null | now()
 updated_at | timestamp with time zone    | not null | now()
```

---

### Step 6 — Configure environment variables

```bash
cp .env.example .env
```

Edit `.env` with your actual values:

```env
PORT=8080
GIN_MODE=debug
JWT_SECRET=your-super-secret-key-here
TOKEN_TTL=24h

DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=your-db-password
DB_NAME=gin_auth
DB_SSLMODE=disable
```

Export the variables into your shell (Go reads from env, not from .env files directly):

```bash
export $(cat .env | xargs)
```

Or use direnv to auto-load on `cd`:

```bash
# Install direnv, then:
echo "dotenv" > .envrc
direnv allow
```

---

### Step 7 — (Optional) Regenerate sqlc code

The `db/sqlc/` folder is already pre-generated and committed. Only re-run this if you change `db/queries/users.sql` or `db/migrations/`:

```bash
sqlc generate
```

---

### Step 8 — Run the server

```bash
go run ./cmd/server
```

Expected output:

```
[GIN-debug] [WARNING] Creating an engine instance with the Logger and Recovery middleware already attached.
[GIN-debug] GET    /health
[GIN-debug] POST   /api/v1/auth/register
[GIN-debug] POST   /api/v1/auth/login
[GIN-debug] GET    /api/v1/auth/profile
Connected to PostgreSQL
Server listening on :8080 (mode: debug)
```

## Testing the API

### Health check

```bash
curl http://localhost:8080/health
```

Response:
```json
{"status": "ok"}
```

---

### Register a new user

```bash
curl -X POST http://localhost:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "alice",
    "email": "alice@example.com",
    "password": "secret123"
  }'
```

Response:
```json
{
  "success": true,
  "message": "user registered successfully",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
      "username": "alice",
      "email": "alice@example.com",
      "created_at": "2024-01-01T00:00:00Z"
    }
  }
}
```

### Get profile (protected route)

Copy the token from the login response and use it here:

```bash
curl http://localhost:8080/api/v1/auth/profile \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

Response:
```json
{
  "success": true,
  "message": "profile retrieved",
  "data": {
    "id": 1,
    "username": "alice",
    "email": "alice@example.com",
    "created_at": "2024-01-01T00:00:00Z"
  }
}
```

---

### Error responses

Wrong password:
```json
{
  "success": false,
  "message": "invalid email or password"
}
```

Missing token:
```json
{
  "success": false,
  "message": "authorization header required"
}
```

Duplicate email:
```json
{
  "success": false,
  "message": "email already registered"
}
```

## Adding a new query (sqlc workflow)

1. Add the SQL query to `db/queries/users.sql`:

```sql
-- name: DeleteUser :exec
DELETE FROM users WHERE id = $1;
```

2. Regenerate the Go code:

```bash
sqlc generate
```

3. A new `DeleteUser(ctx, id)` method is now available on `*db.Queries` in `db/sqlc/users.sql.go`.

4. Use it in the repository layer — no other files need to change.

## Docker Commands

All Docker operations are available via Makefile for convenience:

| Command | Description |
|---------|-------------|
| `make docker-up` | Start all services (app, database, migrations) |
| `make docker-down` | Stop and remove containers |
| `make docker-logs` | View logs from all services |
| `make docker-clean` | Remove containers and reset database volume |
| `make docker-shell-postgres` | Connect to PostgreSQL shell |

### Running new migrations in Docker

When you add new migration files to `db/migrations/`:

**Option 1: Restart with clean database**
```bash
make docker-clean
make docker-up
```

**Option 2: Just restart migrations**
```bash
docker-compose up migrate
```

## Logging

The application logs important events:

- **[INFO]** - Register/login attempts and successes
- **[WARN]** - Validation failures, email conflicts, invalid credentials
- **[ERROR]** - Internal errors (database, hashing, etc.)

Example log output:
```
[INFO] register attempt: email=user@example.com username=john
[INFO] register successful: user_id=123e4567-e89b-12d3-a456-426614174000 email=user@example.com
[INFO] login attempt: email=user@example.com
[INFO] login successful: user_id=123e4567-e89b-12d3-a456-426614174000 email=user@example.com
```
